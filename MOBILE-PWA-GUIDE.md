## 📱 什么是 PWA？

PWA (Progressive Web App) 是一种现代 Web 应用技术，让网站可以像原生 App 一样使用。

### GTD Focus 的 PWA 特性
- ✅ **离线使用** - 无网络时也能查看和编辑任务
- ✅ **添加到主屏幕** - 像原生 App 一样启动
- ✅ **全屏显示** - 无浏览器地址栏
- ✅ **快速加载** - 缓存加速
- ✅ **自动更新** - 后台静默更新

## 🚀 安装到手机

### iOS (iPhone/iPad)

#### 步骤:
1. **打开 Safari**
   - 访问：https://noelyang.github.io/gtd_app/

2. **点击分享按钮**
   - Safari 底部中间的方框 + 箭头图标

3. **向下滑动**
   - 找到 "添加到主屏幕" 选项

4. **点击"添加到主屏幕"**
   - 可以修改应用名称（可选）
   - 点击右上角 "添加"

5. **完成！**
   - 主屏幕出现 GTD 图标
   - 点击图标全屏打开

#### 特点:
- 📱 图标出现在主屏幕
- 🖥️ 点击图标全屏打开
- ❌ 无 Safari 浏览器界面
- ✅ 支持离线使用
- 🎨 支持深色/浅色主题

## 📡 离线功能

### 可用的功能（离线）
- ✅ 查看所有已加载的任务
- ✅ 添加新任务
- ✅ 编辑任务
- ✅ 删除任务
- ✅ 完成任务
- ✅ 切换分类
- ✅ 搜索过滤
- ✅ 切换主题（深色/浅色）

### 需要网络的功能
- ⚠️ WebDAV 同步（上传/下载）
- ⚠️ 首次加载应用

### 离线工作原理
1. **首次访问** - 缓存核心文件（HTML/CSS/JS）
2. **离线使用** - 从缓存加载，数据存本地
3. **恢复网络** - 自动更新缓存
4. **手动同步** - 点击 Sync 同步到云端

## 💾 数据存储

### 存储位置
- **任务数据:** localStorage（浏览器本地存储）
- **配置信息:** localStorage（主题、语言偏好）
- **缓存文件:** Cache Storage（Service Worker）

### 存储限制
| 浏览器 | 存储限制 |
|--------|----------|
| Safari (iOS) | ~1GB |
| Chrome (Android) | ~6GB |

### 数据持久化
- ✅ 关闭浏览器 - 数据保留
- ✅ 重启设备 - 数据保留
- ✅ 更新应用 - 数据保留
- ⚠️ 清除缓存 - 数据丢失
- ⚠️ 卸载应用 - 数据可能丢失

**建议:** 定期使用 WebDAV 同步备份数据

## 🔄 更新机制

### 自动更新
1. **检测更新** - Service Worker 检查新版本
2. **后台下载** - 静默下载新文件
3. **等待激活** - 关闭所有标签页后激活
4. **下次打开** - 使用新版本

### 手动更新
1. 下拉刷新页面
2. 或强制刷新（Ctrl+F5 / Cmd+Shift+R）

### 版本检查
应用会在后台自动检查更新，无需手动操作。

## 🐛 常见问题

### Q1: 为什么安装后还是浏览器界面？
**A:** 某些浏览器/系统不支持 PWA，或安装方式不正确。

**确保:**
- 使用 Safari (iOS) 或 Chrome (Android)
- 按照正确步骤安装
- 系统版本较新（iOS 11.3+ / Android 8+）

### Q2: 离线后无法使用？
**A:** 检查:
- 是否首次访问（需先联网加载）
- Service Worker 是否注册成功
- 浏览器是否支持 PWA

**解决:**
1. 先联网打开应用
2. 等待完全加载
3. 添加到主屏幕
4. 测试离线功能

### Q3: 数据会丢失吗？
**A:** 正常情况不会。但建议:
- 定期同步到 WebDAV
- 不要清除浏览器数据
- 导出重要数据备份

### Q4: 更新后数据没了？
**A:** PWA 更新不会影响 localStorage 数据。

**如果数据丢失:**
- 检查是否清除了浏览器数据
- 从 WebDAV 恢复备份
- 检查浏览器隐私模式

### Q5: 桌面版和移动版数据同步？
**A:** PWA 本身不同步数据，需使用 WebDAV:
1. 设备 A 上传到 WebDAV
2. 设备 B 从 WebDAV 下载
3. 数据保持一致

## 🔧 开发者信息

### Service Worker
- **文件:** service-worker.js
- **策略:** 网络优先，缓存备用
- **缓存:** 核心文件 + 用户访问过的资源

### 缓存策略
```javascript
// 网络优先
fetch(request)
  .then(networkResponse => {
    // 更新缓存
    return networkResponse;
  })
  .catch(() => {
    // 返回缓存
    return cache.match(request);
  });
```
