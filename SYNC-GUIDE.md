# WebDAV 同步使用指南

**版本:** 1.0  
**日期:** 2026-03-02

---

## 📖 什么是 WebDAV 同步？

WebDAV 同步功能允许你将 GTD Focus 的任务数据同步到任意支持 WebDAV 协议的云存储服务，实现多设备间的数据同步。

## 🚀 快速开始

### 步骤 1: 打开同步设置

1. 点击右上角 **Sync** 按钮
2. 进入同步配置界面

### 步骤 2: 配置 WebDAV

填写以下信息:

```
WebDAV Server: https://dav.example.com
Username: your_username
Password: your_password
```

**常见服务的服务器地址:**

#### iCloud Drive
```
服务器：https://dav.icloud.com
用户名：你的 Apple ID
密码：应用专用密码 (需在 appleid.apple.com 生成)
```

#### Nextcloud
```
服务器：https://your-nextcloud.com/remote.php/dav/files/username
用户名：你的 Nextcloud 用户名
密码：你的 Nextcloud 密码
```

#### Synology NAS
```
服务器：http://your-nas-ip:5005
用户名：你的 NAS 用户名
密码：你的 NAS 密码
```

### 步骤 3: 测试连接

点击 **Test Connection** 按钮

- ✅ 绿色提示 "Connection successful!" 表示成功
- ❌ 红色提示 "Connection failed" 表示失败，请检查配置

### 步骤 4: 保存配置

点击 **Save Config** 保存配置

---

## 📤 上传数据到云端

1. 点击 **Upload to Cloud** 按钮
2. 等待上传完成
3. 看到 "Upload successful" 提示

**上传的内容:**
- 所有任务数据 (JSON 格式)
- 文件名：`gtd-backup.json`
- 自动添加时间戳

---

## 📥 从云端下载数据

1. 点击 **Download from Cloud** 按钮
2. 系统自动检测云端和本地数据版本
3. 如果云端数据更新，会提示是否覆盖

**冲突处理:**
- 云端数据更新 → 提示是否覆盖本地
- 本地数据更新 → 提示"本地数据已是最新"
- 同时修改 → 最新时间戳获胜

## 🔄 同步策略

### 手动同步 (推荐)
- 每次更换设备时手动同步
- 上传：本地 → 云端
- 下载：云端 → 本地

### 自动备份 (可选)
- 每次修改后自动上传 (开发中)
- 定期从云端下载 (开发中)

### 数据加密
- ✅ HTTPS 传输加密
- ⚠️ 数据存储加密 (开发中)
- ✅ 密码本地存储 (localStorage)

- 数据存储在用户自己的云盘

- 可随时删除云端备份

### 最佳实践
1. 使用强密码
2. 启用两步验证 (如支持)
3. 定期导出本地备份
4. 不要在公共电脑使用

## ⚠️ 常见问题

### Q1: 连接失败怎么办？

**检查清单:**
- [ ] 服务器地址是否正确 (包含 https://)
- [ ] 用户名密码是否正确
- [ ] 网络是否正常
- [ ] 服务器是否可访问

**解决方案:**
1. 检查服务器地址格式
2. 重新生成应用专用密码 (如 iCloud)
3. 检查防火墙设置
4. 联系云服务提供商

### Q2: 上传失败怎么办？

**可能原因:**
- 云盘空间不足
- 文件权限问题
- 网络中断

**解决方案:**
1. 检查云盘剩余空间
2. 手动创建 `gtd-backup.json` 文件测试权限
3. 重试上传

### Q3: 下载失败怎么办？

**可能原因:**
- 云端没有备份文件
- 文件格式错误
- 网络问题

**解决方案:**
1. 先上传一次数据
2. 检查云端文件是否存在
3. 检查网络连接

### Q4: 多设备同时修改怎么办？

**冲突解决机制:**
- 系统自动比较时间戳
- 最新的修改获胜
- 覆盖前会提示确认

**最佳实践:**
- 在固定设备上编辑
- 编辑前先同步
- 重要数据定期导出备份

## 📱 多设备同步流程

### 场景：手机 ↔ 电脑 同步

#### 在电脑上:
1. 打开 GTD Focus
2. 配置 WebDAV (首次)
3. 点击 **Upload to Cloud**

#### 在手机上:
1. 打开 GTD Focus
2. 配置相同的 WebDAV
3. 点击 **Download from Cloud**

#### 之后每次:
- 电脑编辑后 → 上传
- 手机编辑后 → 上传
- 切换设备时 → 下载

## 💾 数据格式

### 云端备份文件结构

```json
{
  "version": 1,
  "timestamp": 1709323200000,
  "data": [
    {
      "id": "1234567890",
      "title": "Task Title",
      "description": "Description",
      "category": "inbox",
      "completed": false,
      "tags": [],
      "createdAt": "2026-03-02T12:00:00.000Z",
      "updatedAt": "2026-03-02T12:00:00.000Z"
    }
  ]
}
```

### 手动恢复数据

如果需要同步失败，可以手动导入:

1. 从云端下载 `gtd-backup.json`
2. 在 GTD Focus 中点击 **Data** → **Import**
3. 选择文件导入

## 🛠️ 高级配置

### 自定义备份路径

修改代码中的备份文件名:

```javascript
// 默认
const filename = 'gtd-backup.json';

// 自定义
const filename = 'my-tasks/backup.json';
```

### 启用自动同步 (开发中)

```javascript
// 每次保存后自动上传
app.saveTasks = function() {
    localStorage.setItem('gtd-tasks', JSON.stringify(this.tasks));
    this.uploadToWebDAV(); // 自动上传
    this.render();
};
```

## 🆘 获取帮助

### 遇到问题？

1. **检查配置** - 确认 WebDAV 配置正确
2. **查看日志** - 打开浏览器控制台 (F12) 查看错误信息
3. **导出备份** - 先导出本地数据防止丢失
4. **联系支持** - 提供错误信息和截图

### 错误代码

| 错误 | 含义 | 解决方案 |
|------|------|----------|
| 401 | 认证失败 | 检查用户名密码 |
| 403 | 权限不足 | 检查文件权限 |
| 404 | 路径不存在 | 检查服务器地址 |
| 500 | 服务器错误 | 联系服务提供商 |
| Timeout | 连接超时 | 检查网络连接 |
